# Issue-27: Agent 작업 수행 전 체크리스트 엄수

2025-01-15

## 작업 내용

### 목표
Agent가 작업을 시작하기 전에 반드시 확인하고 준수해야 하는 체크리스트를 정의하고, 이를 통합 가이드 문서로 작성하여 작업의 품질, 효율성, 추적성을 보장함.

1. **체크리스트 항목 정의**
   - 체크리스트 1: Agent 컨텍스트 설정 (충분성과 최소성)
   - 체크리스트 2: DAG 의존성 그래프 기반 작업 수행
   - 체크리스트 3: NF-REQ 및 Traceability 추적성 보강

2. **통합 체크리스트 가이드 문서 작성**
   - `docs/AGENT_PRE_WORK_CHECKLIST.md` 파일 생성
   - 각 체크리스트 항목별 상세 가이드 작성
   - 통합 체크리스트 및 작업 시작 프로세스 정의

3. **관련 이슈 통합**
   - Issue 24: Agent 컨텍스트 설정의 충분성과 최소성 → 체크리스트 1로 통합
   - Issue 25: Agent 병렬 실행 구조를 DAG 의존성 그래프로 먼저 표현 → 체크리스트 2로 통합
   - Issue 26: Traceability를 작업전에 완전히 구조화하면 더 좋겠음 → 체크리스트 3으로 통합

## 체크리스트 항목 상세

### 체크리스트 1: Agent 컨텍스트 설정

**목적**: Agent에 충분하면서도 중복이 없는 컨텍스트 설정

**주요 확인 사항**:
- 관련 규칙 파일 확인 (`.cursor/rules/` 디렉토리)
- 관련 문서 확인 (`docs/` 디렉토리)
- 기존 코드베이스 분석
- 중복 작업 확인
- 컨텍스트 최소화

**참고**: Issue 24에서 정의된 컨텍스트 설정의 충분성과 최소성 원칙 적용

### 체크리스트 2: DAG 의존성 그래프 기반 작업 수행

**목적**: 의존성 그래프 DAG 작성을 기반으로 Agent 작업 수행

**주요 확인 사항**:
- `docs/BE_ISSUE_DAG.md` 파일 확인
- 선행 이슈 완료 여부 확인
- 병렬 실행 가능 여부 확인
- 공통 리소스 충돌 가능성 확인
- Critical Path 우선순위 확인

**참고**: Issue 25에서 작성된 DAG 의존성 그래프 활용

### 체크리스트 3: NF-REQ 및 Traceability 추적성 보강

**목적**: NF-REQ 및 Traceability 문서 기반의 간접적인 추적성도 Issue에 보강

**주요 확인 사항**:
- `docs/SRS/SRS_V0.3.md`의 NF-REQ 섹션 확인
- 관련 기능 요구사항(REQ-FUNC-XXX) 확인
- 관련 비기능 요구사항(REQ-NF-XXX) 확인
- PRD의 기능 요구사항(F1~F10)과의 매핑 확인
- Issue 문서에 추적성 정보 추가

**참고**: Issue 26에서 정의된 Traceability 구조화 원칙 적용

## 활용 방법

1. **Agent 작업 시작 전**
   - `docs/AGENT_PRE_WORK_CHECKLIST.md` 파일 참조
   - 통합 체크리스트의 모든 항목 확인
   - 작업 시작 프로세스에 따라 단계별 진행

2. **체크리스트 확인 순서**
   - Step 1: 컨텍스트 수집
   - Step 2: 의존성 확인
   - Step 3: 요구사항 추적성 확인
   - Step 4: 작업 시작

3. **Issue 문서 작성 시**
   - 체크리스트 3에 따라 Issue 문서에 추적성 정보 추가
   - 관련 REQ-FUNC-XXX, REQ-NF-XXX 명시
   - 관련 User Story 및 PRD 섹션 참조 추가

## 주요 성과

- 요구사항 변경 시 영향도 분석 용이

### 작업 품질 보장
- 컨텍스트 설정의 충분성과 최소성 원칙 적용
