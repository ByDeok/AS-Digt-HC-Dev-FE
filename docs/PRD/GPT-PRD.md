Product Requirement Document (v0)

1. 목적 및 목표
  북극성 KPI / 보조 KPI
  제품 비전 요약

  - 제품 비전: “시니어·보호자 이중 UX와 의료급 신뢰 포맷을 갖춘 상호운용 허브로, 3분 온보딩에서 시작해 의사용 1장 요약과 ‘오늘 3가지 행동’을 통해 진료 수용성과 일상 순응을 동시에 향상한다.”

  - 북극성 KPI
    - 외래 현장 ‘의사용 1장’ 리포트 채택률
      - 기준선(파일럿 T0 가정): 10%
      - 목표: 40%+
      - 측정 주기/창구: 주간·월간 / 리포트 뷰·EMR 첨부 이벤트 로그(대시보드)

  - 보조 KPI
    - 온보딩 완료율(End-to-End 3분 플로우)
      - 기준선(가정): 35%
      - 목표: 65%+
      - 측정: 퍼널 단계별 완료/이탈 로그, p50 완료 시간 ≤ 180초
    - 첫날 핵심행동 도달률(‘오늘 1~3개’ 카드 클릭→완료)
      - 기준선(가정): 30%
      - 목표: 60%+
      - 측정: 앱 이벤트(카드 노출→클릭→완료), D1·W1 코호트
    - 월간 ‘의사용 1장’ 생성률(활성이용자 대비)
      - 기준선(가정): 20%
      - 목표: 50%+
      - 측정: 월간 고유 사용자 대비 리포트 생성 이벤트
    - 가족 대리접근 설정률
      - 기준선(가정): 20%
      - 목표: 50%+
      - 측정: 초대→동의→역할부여 완료 이벤트
    - 90일 유지율
      - 기준선(가정): 20%
      - 목표: 35%+
      - 측정: D90 코호트 유지율(월간 활성 기준)
    - 복약 순응도 향상
      - 기준선: 파일럿 T0 측정 예정
      - 목표: +15%p
      - 측정: 약 복용 완료율(전자 이벤트/자가보고) 비교
    - 의사결정 시간(재내원 계획/추가검사) 단축
      - 기준선: 파일럿 T0 측정 예정
      - 목표: -20%
      - 측정: 외래 현장 설문/타임스탬프(리포트 열람→오더 입력)

2. 사용자 시나리오 및 JTBD
  Given–When–Then 형식
  Acceptance Criteria (3개 이상, 수치 포함)

  - Story 1: As a Senior, I want a one-page doctor summary so that I can present trustworthy, concise health information in clinic.
    - AC1 (성능·완결성): Given 최근 6개월 데이터가 연동됨, When 1장 요약을 생성하면, Then p95 생성 시간 ≤ 3초이고 주요 지표(활동·심박·혈압·체중)의 기간·기기·단위가 표준화되어 표기된다(필드 완결성 ≥ 95%).
    - AC2 (현장 활용성): Given 외래 대기 중, When 리포트를 열람하면, Then PDF·인쇄 가능하며 EMR 첨부까지 3클릭 이내, 실패율 < 0.5%.
    - AC3 (가독성·접근성): Given 접근성 모드 활성, When 리포트를 열람하면, Then 대글자/고대비/스크린리더 레이블 누락 0건, 문맥 설명 툴팁 커버리지 ≥ 90%.
    - AC4 (신뢰): Given 측정 맥락/기기 정보가 존재, When 리포트를 생성하면, Then 각 지표에 기간·기기·결측 태그가 표시된다(누락율 < 5%).

  - Story 2: As a Senior, I want only 1–3 actionable nudges per day so that I can reduce fatigue and build consistent habits.
    - AC1 (정확·과다알림 방지): Given 알림 창구 설정, When 행동 카드가 생성되면, Then 일일 제안 개수 1~3개, 야간(22~07시) 자동 침묵, 오경보율 < 2%.
    - AC2 (완료 품질): Given 행동 카드가 노출, When 사용자가 수락/완료하면, Then D1 완료율 ≥ 25%, W1 주간 3일 이상 달성률 ≥ 50%.
    - AC3 (응답성): Given 앱 포그라운드, When 행동 카드를 열면, Then p95 응답 ≤ 800ms, 실패율 < 0.5%.
    - AC4 (실패 케이스): Given 네트워크 장애, When 동기화 실패 시, Then 재시도(지수백오프) 및 오프라인 큐잉으로 데이터 유실 0건.

  - Story 3: As a Caregiver, I want a shared family board with delegated roles so that we can coordinate schedules, meds, and risks together.
    - AC1 (역할·권한): Given 초대가 발송됨, When 보호자가 수락하면, Then 대리접근·역할(Viewer/Editor/Admin) 설정이 3단계 내 완료, 권한 템플릿 가이드 제공.
    - AC2 (동기화): Given 보드에 일정/약이 변경, When 다른 구성원이 열람하면, Then 변경 전파 지연 ≤ 60초, 충돌 시 최신기준 병합·감사로그 기록.
    - AC3 (프라이버시): Given 민감정보 접근, When 비정상 위치/시간 접근이 탐지되면, Then 위험 기반 추가 인증(OTP/바이오) 요구, 실패 시 알림 및 접근 차단.
    - AC4 (실패 케이스): Given 초대 링크 만료, When 재요청하면, Then 새 링크 발급까지 1클릭, 실패율 < 1%.

  - Story 4: As a New user, I want to complete onboarding in under 3 minutes so that I can reach first value on day one.
    - AC1 (시간·완료): Given 신규 설치, When 가입→인증→권한→디바이스/병원 연동을 진행하면, Then p50 완료 시간 ≤ 180초, 전체 성공률 ≥ 65%.
    - AC2 (퍼널 품질): Given 단계 전환, When 각 단계를 완료하면, Then 단계별 이탈률 ≤ 15%, 되돌아가기/중간저장 제공.
    - AC3 (접근성): Given 접근성 설정 적용, When 온보딩 화면 탐색 시, Then 스크린리더 라벨 완전성 100%, 포커스 트랩 없음, 폼 오류 힌트 노출율 100%.
    - AC4 (예외): Given 병원 포털 미지원 지역, When 연동을 선택하면, Then 파일 업로드 대체 경로 제공(가이드 포함), CS 티켓 생성까지 2클릭.

3. Pain / Needs 정의
  실패 KPI 수치화 포함

  - Core 시니어
    - Pain: 분절된 데이터·채널로 진료 전 설명 부담, 복약/검사 일정 누락
      - 실패 KPI 예: 진료 전 자료 정리 시간 > 3분 사용자가 70%+, 지난 30일 내 복약 누락 ≥ 2회 사용자 30%+, 검사 미이행 ≥ 1건 사용자 20%+
    - Needs: “의사와 같은 그림” 1장 요약, 다음 행동 제시, 병원/검진 자동 연동

  - 보호자(샌드위치 세대)
    - Pain: 부모 알림·일정·결제가 다채널, 권한·역할 혼선
      - 실패 KPI 예: 가족 보드/권한 미설정 사용자 80%+, 일정 충돌·중복 알림 월 2회+
    - Needs: 대리접근/역할 분담이 가능한 가족 보드, 권한 템플릿 단순화

  - Extreme/Low-Digital 시니어
    - Pain: 작은 글씨·저조한 대비·스크린리더 미지원, 장황한 온보딩
      - 실패 KPI 예: 접근성 모드 미탐지/레이블 누락 > 5건/화면, 온보딩 p50 > 6분
    - Needs: 3분 온보딩, 대글자/고대비/음성 요약, 최소 행동 중심 알림(야간 침묵·오경보 최소화)

4. 기능 요구사항 (MSCW 분류)
  Must / Should / Could / Won’t
  각 기능의 근거 및 의존성

  - Must
    - F1. 3분 온보딩 일괄 흐름(가입·인증·권한·연동) — 근거: AOS/DOS Top, Q1 트리플렛(온보딩) — 의존성: OAuth/기관 포털, 디바이스 SDK, 동의/위임 모듈
    - F2. 1장 의사용 요약 리포트 — 근거: VPS Outcome Top, 진료 전 데이터 정리·공유 Job — 의존성: 데이터 통합·표준화(FHIR/PHR 게이트웨이), PDF/인쇄, EMR 첨부
    - F3. 오늘 1~3개 행동 코칭 — 근거: VPS Outcome Top, 행동 변화 코칭 Job — 의존성: 목표 엔진, 알림 스케줄러, 조용시간/오경보 제어
    - F4. 가족 공유 보드/대리 접근 — 근거: VPS MVP Feature Map(F4 High/MVP), 보호자 협업 Job — 의존성: 초대·동의·역할 템플릿, 감사로그
    - F5. 병원 포털/검진 1곳 연동 — 근거: VPS MVP Feature Map(F5 High/MVP), 의료급 데이터 신뢰 — 의존성: 포털 API/스크래핑 허브, 보안 게이트웨이
    - F10. 접근성 퍼스트(A11y 패키지) — 근거: VPS MVP Feature Map(F6 High/MVP), Extreme 시니어(E1/E2) — 의존성: 폰트/대비 토큰, 스크린리더 라벨링
    - F7. 의사 신뢰 포맷(맥락/기기/기간 표기) — 근거: 임상 수용성, “의료급 신뢰 포맷” 차별 가치 — 의존성: 메타데이터 스키마, 결측 태깅

  - Should
    - F8. 권한·프라이버시 단순화 — 근거: 다채널 혼선 — 의존성: 위험 기반 인증, 세션·권한 정책

  - Could
    - F6. 멀티디바이스 정합성(워치/혈압계 2종) — 근거: 데이터 신뢰 — 의존성: 커넥터 프레임워크, 동기화 규칙
    - F9. 복약 누락 방지(톤다운/야간침묵) — 근거: 알림 피로 저감 — 의존성: 알림 룰 엔진

  - Won’t (초기 범위 제외)
    - 완전한 EMR 양방향 연동, 보험 청구/심사, 의료진 대시보드 정식출시(파일럿 이후)

5. 비기능 요구사항 (NFR)
  성능, 보안, 가용성, 비용 등
  임계치, 측정 기준, 알림 조건 포함

  - 성능
    - 앱 초기 로드 p95 ≤ 1.5s, 주요 화면 전환 p95 ≤ 800ms
    - 1장 리포트 생성 p95 ≤ 3s, PDF 렌더 p95 ≤ 2s
  - 신뢰성/가용성
    - 월 가용성 ≥ 99.5%, 백엔드 오류율 < 0.5%
    - 데이터 동기화 지연 p95 ≤ 60s(가족 보드·알림)
  - 보안/프라이버시
    - 최소 권한·데이터 지역성·전송/저장 암호화(AES-256/ TLS1.2+)
    - 위험 기반 인증(고위험 시 2FA), 동의/위임/감사로그 전 항목 기록
  - 비용
    - 사용자당 월 인프라비용(유효 DAU 기준) ≤ $0.25, 데이터 저장 단가 월 ≤ $0.05/사용자
  - 모니터링/알림
    - 대시보드: 지연/오류/가용성·비용 지표 실시간
    - 알림: 임계치 초과 5분 내 온콜, 반복 초과 시 에스컬레이션

6. Differential Value
  대안 대비 수치 비교 (성능·정확·비용 중 2개 이상)

  - 병원 방문 전 수기 정리/구두 설명 대비
    - 진료 전 자료 정리 시간 -40% (5분 → 3분), 의사 이해도/결정 속도 +20%
  - 빅테크 건강앱 대비
    - 의료급 포맷(맥락/기기/기간·결측 태그) 완결성 +30%p, EMR 워크플로 부합성(첨부·인쇄) 3클릭 이내
  - 국내 B2B2C 패키지 대비
    - 시니어 원스텝 퍼널로 온보딩 완료율 +25~35%p, 가족 대리접근 설정률 +20~30%p
  - 비용 효율
    - 동일 DAU 기준 인프라 비용 -20%(캐싱·스트리밍 최적화) — 실험으로 검증

7. Proof & Experiment
  실험 설계 및 측정 항목 연결

  - 실험 1: 온보딩 3분 플로우 A/B
    - 디자인: 기존 플로우 대비 3분 일괄 플로우
    - 표본: n ≥ 1,000(신규), 기간: 2주
    - KPI: 완료율(+25~35%p), p50 완료 시간 ≤ 180초, 단계별 이탈률 ≤ 15%
    - 측정: 퍼널 로그, 리텐션 D1
    - 성공 기준: 완료율 +25%p 이상 또는 p50 180초 이하 만족

  - 실험 2: ‘오늘 1~3개 행동’ 카드
    - 디자인: 알림 톤다운/야간침묵 + 1~3개 제안 vs 기존
    - 표본: n ≥ 1,000(활성), 기간: 4주
    - KPI: D1 완료율 ≥ 25%, W1 3일 달성률 ≥ 50%, 오경보율 < 2%
    - 측정: 이벤트 로그, 알림 반응, 사용자 설문(피로도)
    - 성공 기준: 완료·달성·오경보 기준 동시 충족

  - 실험 3: 1장 요약 리포트 현장 채택
    - 디자인: 파일럿 병원 1곳, 리포트 유무 랜덤 배정(세션 단위)
    - 표본: n ≥ 500(세션), 기간: 4주
    - KPI: 외래 현장 채택률 40%+, 의사/환자 평점(5점) 4.2+, 의사결정 시간 -20%
    - 측정: 리포트 뷰·EMR 첨부 로그, 설문, 타임스탬프
    - 성공 기준: 채택률/평점/시간 단축 중 2개 이상 달성

8. 리스크 및 가정 (ADR)
  리스크 3개 이상, 검증 계획 포함

  - 리스크
    - 병원/검진 포털 연동 범위 편차 → 단계적 파일럿, 커넥터 추상화, 수동 업로드 대체
    - Extreme 접근성 구현 난이도 → 접근성 항목을 P1 버그로 관리, 사전 점검표·자동 테스트
    - 데이터 신뢰의 임상 수용성 → 맥락/기기/기간 표준 표기, 임상 코디자인·벤치마크
  - 가정
    - Q1 트리플렛(3분 온보딩·행동 코칭·1장 요약)이 도입/재사용 임계치를 만든다
    - B2B2C 채널에서 시니어 원스텝 퍼널이 기관 병목을 대체할 수 있다
  - 검증 계획(ADR 링크 예정)
    - 파일럿 범위·의사결정 기록(ADR), 실패/후퇴 기준 사전 명시

9. 범위 정의
  In / Out of Scope 명시

  - In Scope
    - 3분 온보딩(가입·인증·권한·연동), 1장 의사용 요약, 오늘 1~3개 행동
    - 가족 공유 보드/대리 접근, 병원 포털 1곳 연동, 접근성 패키지
  - Out of Scope(초기)
