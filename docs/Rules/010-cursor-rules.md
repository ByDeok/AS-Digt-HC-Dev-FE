# .mdc 파일 구조 가이드
- 파일 명명: NNN-name.mdc (NNN = 001–299)
  - 001–099: 핵심/프로젝트 전역 규칙
  - 100–199: 워크플로우 및 통합 규칙
  - 200–299: 패턴 및 스타일 규칙
  - 300–399: 기술별 규칙 (프레임워크, 라이브러리, 도구)
- Glob이 겹칠 때 더 높은 NNN이 우선순위를 가짐
- 각 규칙은 단일 논리적 목적을 대상으로 해야 함 (SRP)
- Cursor GUI를 사용하여 규칙 유형 할당: Always, Auto, Manual, AgentRequested
- 본문을 간결하게 유지: 이상적으로 25줄 이하
