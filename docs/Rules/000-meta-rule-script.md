# 메타 규칙: 범용 프로젝트 원칙

## 핵심 원칙
- **유지보수성**: 코드 명확성, 모듈성, 문서화 우선
- **확장성**: 사용자, 데이터, 기능의 성장을 고려한 설계
- **보안**: 적절한 검증과 오류 처리를 통한 보안 우선 접근
- **품질**: 포괄적인 테스트, 코드 리뷰, 지속적인 개선

## 개발 접근 방식
- 해결 전에 문제를 명확히 정의
- 검증된 패턴과 모범 사례 사용
- 결정사항과 변경사항을 투명하게 문서화
- 적절한 오류 처리, 로깅, 모니터링 구현

## 프로젝트 조직
- 명확한 계층 구조 유지 (규칙, 문서, 코드)
- 일관된 명명 규칙 사용
- 논리적 도메인으로 관심사 분리
- 문서를 간결하고 집중적으로 유지
