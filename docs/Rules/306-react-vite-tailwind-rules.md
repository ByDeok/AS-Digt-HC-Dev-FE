- 확인한 후 코드를 작성하세요!

- 성능보다 쉬움과 가독성에 집중하세요.

### 프로젝트 설정 우선순위
1. **기본값: Vite + React** (정적 배포 가능)
   - `npm create vite@latest my-app -- --template react-ts`
2. **피하기**: create-react-app (성능 및 유지보수 문제)

### 코딩 환경
사용자는 다음 코딩 언어에 대해 질문합니다:
- ReactJS
- JavaScript
- TypeScript
- TailwindCSS
- HTML
- CSS
아래 코딩 환경 규칙이 더 중요합니다:
- HTML 요소 스타일링에는 항상 Tailwind 클래스를 사용하세요; CSS나 태그 사용을 피하세요.
- 함수 대신 const를 사용하세요. 예: "const toggle = () =>". 가능하면 타입도 정의하세요.

### 코드 구현 가이드라인
코드를 작성할 때 다음 규칙을 따르세요:
- 가능한 경우 조기 반환을 사용하여 코드를 더 읽기 쉽게 만드세요.
- HTML 요소 스타일링에는 항상 Tailwind 클래스를 사용하세요; CSS나 style 태그를 피하세요.
- 클래스 태그에서 가능한 경우 삼항 연산자 대신 "class:"를 사용하세요.
- 설명적인 변수 및 함수/const 이름을 사용하세요. 또한 이벤트 함수는 "handle" 접두사로 명명해야 합니다. 예: onClick의 경우 "handleClick", onKeyDown의 경우 "handleKeyDown".
- 요소에 접근성 기능을 구현하세요. 예를 들어, 태그는 tabindex="0", aria-label, on:click, on:keydown 및 유사한 속성을 가져야 합니다.
- 함수 대신 const를 사용하세요. 예: "const toggle = () =>". 가능하면 타입도 정의하세요.
